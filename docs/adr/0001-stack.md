# ADR 0001: Выбор технологического стека

- Статус: принято
- Дата: 2026-07-06

## Контекст

Стартуем HEMA-турнирное приложение с нуля. Пет-проект, разрабатываемый
преимущественно нейросетями (AI-first). Нужны: веб-клиент, серверная логика,
БД, чёткие контракты API и воспроизводимая локальная среда.

## Решение

| Слой      | Выбор                                    |
| --------- | ---------------------------------------- |
| Клиент    | Next.js (App Router), TypeScript, pnpm   |
| BFF       | Next.js Route Handlers (Node), REST      |
| Транспорт | REST (browser→BFF), gRPC/Connect (BFF→server) |
| Сервер    | Go 1.24+, Connect-Go                     |
| БД        | PostgreSQL                               |
| Доступ БД | sqlc + pgx + goose                       |
| Контракты | Protobuf + buf → connect-go / connect-es |
| Auth      | JWT (access+refresh), argon2id           |
| Инфра     | docker-compose                           |
| Репо      | монорепо: /server /web /proto /deploy /docs |

## Обоснование

- **Next.js App Router** — актуальный подход, BFF на Route Handlers даёт чёткое
  разделение REST наружу и gRPC внутрь.
- **Connect (go+es)** — единый протокол, совместимый с gRPC, генерируется из
  одного источника proto через buf; отличная типобезопасность на обоих концах.
- **sqlc + pgx** — явный типобезопасный SQL, AI-friendly (запросы читаемы),
  goose — простые версионируемые миграции.
- **argon2id** — современный memory-hard алгоритм хеширования паролей
  (рекомендация OWASP), хранит параметры и соль в самодостаточной PHC-строке.
- **JWT access+refresh** — stateless-аутентификация без внешних зависимостей,
  полный контроль на стороне Go.
- **Монорепо** — единый источник proto, docker-compose на верхнем уровне,
  удобная навигация для агентов.

## Последствия

- Требуются инструменты: `buf`, `sqlc`, `goose`, `pnpm` (ставятся через
  `go tool`/corepack или локально).
- Сгенерированный код (`gen/`, `sqlc/`) не редактируется руками.
- Отзыв refresh-токенов не реализован в v1 (stateless JWT) — вынесено в будущий ADR.
