## Обзор решения

Чисто web-инкремент (`/web`), синхронный, **без** новых модулей, RPC и без
изменения контрактов/BFF. Работаем в FSD-слоях `shared/config`, `shared/ui`,
`shared/lib`, `entities/user/ui`, `widgets`, `features/auth/ui` и в роутах
`src/app/**` (не `app/api/**`) плюс токены темы в `globals.css`. Четыре
направления: (1) выпрямить навигацию — кабинет в меню аватара, отдельная
под-навигация админки, активные пункты; (2) убрать принудительный редирект в
кабинет после логина; (3) переверстать главную (турнир+номинации в центре) и
вынести «о платформе» в отдельный роут `/about`; (4) ввести фирменную палитру
(зелёный primary + золото-акцент) через CSS-токены. Границы импортов FSD
(ADR 0005) и правила тем (ADR 0005 §7) сохраняются.

### 1. Токены палитры — `src/app/globals.css`

Ввести фирменную тему поверх текущей нейтральной базы (правим только
CSS-переменные в `:root`/`.dark` + регистрация в `@theme inline`):

- `--primary` / `--primary-foreground` → **зелёный** в обеих темах (подобрать
  oklch так, чтобы контраст текста ≥ 4.5:1; ориентир — зелёный ~`#1f7a4d`,
  `--primary-foreground` — near-white). Это автоматически перекрасит первичные
  `Button`, фокус-кольца через `--ring` (подстроить `--ring` под зелёный),
  ссылки-`text-primary`.
- Новый семантический акцент-токен **золото**: `--gold` (≈ `#d4af37`) и
  `--gold-foreground` (тёмный, для текста поверх золота). Зарегистрировать в
  `@theme inline` как `--color-gold` / `--color-gold-foreground`. **Не** трогать
  shadcn-`--accent` (он остаётся нейтральным hover-фоном — иначе позолотятся все
  ховеры).
- Проверить `--ring` (фокус) — увязать с зелёным, но сохранить видимость в обеих
  темах. `--destructive` не трогаем.
- Обе темы: убедиться, что зелёный/золото читаемы на светлой и тёмной базе (AC-8,
  FR-12).

- Опционально вынести пункты меню аватара / под-навигации админки в конфиг, если
  это упрощает переиспользование (иначе держать локально в соответствующих
  виджетах). Решение — на этапе реализации; главное — единый источник для
  публичного меню.
- Обновить/сохранить `site-config.test.ts`: инвариант «нет пунктов в никуда»

### 4. Хелпер активного пункта — `src/shared/lib/`

### 5. Меню профиля (аватар) — `src/widgets/navbar/user-menu.tsx`

  `UserMenu` уже клиентский; роль пользователя приходит из `user` (проп из

- `Navbar` (`widgets/navbar/navbar.tsx`): пункт «Админка» из основного ряда

  только публичным (турнир/номинации/о платформе). Пункты навигации — рендерить

### 9. Под-навигация админки — `src/app/(admin)/**`

- Ввести под-навигацию админ-зоны как отдельный ряд/панель, видимый на всех

- Пункты: **Пользователи** (`/admin`), **Турнир** (`/admin/tournament`),

- Убрать дублирующую навигацию из `AdminHeader.actions` на `/admin` (кнопки

  упростить/убрать, т.к. под-навигация даёт возврат к разделам. Заголовки
  под-страниц сохранить.

### 10. Server components vs client / state

- `Navbar`, главная, `/about`, admin-страницы — серверные (данные через
  `getCurrentUser`/`getActiveTournament`/`getNominations`).
- Клиентские: `UserMenu` (уже), `admin-nav` (нужен `usePathname`), публичные
  пункты навигации с подсветкой активного (если делать через `usePathname` —
  вынести в небольшой клиентский компонент `nav-links.tsx`; либо для якорей
  подсветка не критична, а для роутов — клиентский рендер). `AuthDialog` (уже).
- State: изменений в state-менеджменте нет (server-state → RQ, UI-state →
  zustand). Auth-модалка — прежний zustand-store; правим только `onSuccess`.

## Риски и открытые вопросы

- **Подсветка активного пункта для якорей.** Для секций главной (`#tournament`,
  `#nominations`) «активность» по скроллу — избыточна для MVP; ограничиваемся
  подсветкой активного **роута** (публичный пункт «О платформе», под-навигация
  админки). Риск низкий; scroll-spy — вне скоупа.
- **Контраст золота.** Золото светлое: на белой базе как текст/контур требует
  достаточной насыщенности; фон-бейдж — с тёмным текстом. Проверить обе темы
  (FR-11/FR-12). Митигируем подбором oklch и `--gold-foreground`.
- **Клиент/сервер для навигации.** Подсветка активного требует `usePathname`
  (клиент). Держать клиентскую часть минимальной (только ссылки), не утягивая
  в клиент серверные данные (`user` приходит пропом из серверного `Navbar`).
- **Layout админ-зоны.** Ввод `(admin)/admin/layout.tsx` с под-навигацией не
  должен ломать существующий серверный guard в `(admin)/layout.tsx` (остаётся).
  Проверить порядок layout'ов и что guard срабатывает раньше рендера
  под-навигации.
- **Объём и «расползание».** Много мелких правок по разным файлам; держим
  границу — только навигация/поток/палитра, без новых продуктовых фич
  (Вне скоупа).
