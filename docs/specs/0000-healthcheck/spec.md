# Spec: Health-check

> Пример-скелет для демонстрации формата SDD (ADR 0008). Не-доменная фича,
> без предметных допущений. Показывает, как заполнять `spec.md`.

- Статус: ready
- Дата: 2026-07-07

## Проблема

Инфраструктуре (docker-compose healthcheck, оркестратор, мониторинг) нужен
дешёвый способ узнать, что сервер поднялся и готов обслуживать запросы. Сейчас
единственный способ проверить живость — дёрнуть доменный RPC, что требует
валидных данных и лишних зависимостей.

## Акторы

- **Внешняя система** (docker/оркестратор/мониторинг) — периодически опрашивает
  эндпоинт, не аутентифицирована.

## Пользовательские сценарии

1. Как **оркестратор**, я хочу дёрнуть health-эндпоинт без токена, чтобы понять,
   готов ли контейнер принимать трафик.
2. Как **разработчик**, я хочу одной командой проверить, что локальный сервер
   запустился, не создавая пользователя.

## Требования

- FR-1: Сервер отдаёт статус готовности по неаутентифицированному запросу.
- FR-2: Ответ включает признак готовности (`serving`/`not serving`).
- FR-3: Эндпоинт не требует доступа к БД для базового «процесс жив» ответа.
- NFR-1: Ответ быстрый (< 50мс) и не создаёт нагрузки.

## Критерии приёмки

- **AC-1**
  - Given: сервер запущен
  - When: вызывается health-RPC без заголовка `Authorization`
  - Then: возвращается ответ со статусом `SERVING` и кодом `OK`
- **AC-2**
  - Given: сервер запущен, интерсептор auth настроен
  - When: вызывается health-RPC без токена
  - Then: запрос не отклоняется как `Unauthenticated` (эндпоинт публичный)

## Вне скоупа

- Проверки готовности зависимостей (БД, внешние сервисы) — отдельная фича
  (readiness с проверкой пула). Здесь только liveness.
- Метрики/трейсинг.
